# Companies Section Architecture & Profiles

This document details the layout structure, profiling guidelines, and operational division configurations for the Companies section.

---

## 1. Page Purpose & Content Mandate
The [companies.html](file:///Users/aravindsaj/Library/CloudStorage/GoogleDrive-ask5615@g.rit.edu/My%20Drive/DeskEssentials/Systems/Insight%20Website/companies.html) page provides detailed descriptions of Aoura Group's operating divisions. It functions as a structured corporate catalog rather than a marketing pitch.

---

## 2. Division Profile Structure
Each division must be represented on this page using a consistent structure:

### 2.1 Aoura Parent holding
- **Role**: Capital allocation, governance, strategic direction.
- **Positioning**: Focused on long-term value creation.

### 2.2 Aoura Products
- **Positioning**: Wholesale B2B supply partner (not a retail storefront).
- **Core Operations**: Centralized logistics hubs, regional inventory reserves, next-day fleet logistics.
- **Visuals**: Clean, structural data representing warehouse logistics capacities.

### 2.3 Aoura E-commerce Services
- **Positioning**: Marketplace accelerator and logistics coordinator (not a design agency).
- **Core Operations**: Noon and Amazon UAE platform operations, catalog optimization, algorithmic analytics.

---

## 3. Formatting & Styling Rules
- **Profile Sections**: Each company profile must contain sections for: **Overview**, **History/Mandate**, **Core Operations**, and **Synergy/Integration**.
- **Data Callouts**: Use Geist Mono layout highlights to outline operational statistics (e.g., number of active SKUs, warehousing square footage, fleet size).
- **Visual Dividers**: High-contrast, 1px horizontal borders (`var(--border-color)`) must separate the divisions to maintain clean visual reading lines.
